Important note: The following content is a work in progress draft of guidance around how to use IBM Plex, IBM’s new corporate typeface. This typeface is open sourced. It can be used by IBM employees and by the public.

We welcome all feedback and improvements through our GitHub issues. All written copy and examples are subject to change. Anyone who uses the font files and code may experience breaking changes at this time.

Typography is the appearance and arrangement of type. Each word or paragraph lives in a grid system—creating helpful alignments, harmonious relationships and clear organizations. Typography should reflect careful choices in size, placement, and weight to create useful hierarchies to guide users through the content.

With our new bespoke corporate typeface, IBM Plex™, comes a new set of guidance and best practices. IBM typography is international and modern to reflect our brand and our design principles.

Style

Typography creates purposeful texture. It guides users to read and understand the hierarchy of information. The right typographic treatment and the controlled usage of type styles helps manage the display of content, keeping it useful, simple and effective.

Weights

We suggest to use IBM Plex Light, Regular, SemiBold, and Italic for digital experiences. The semibold weight is ideal for headers and hierarchies but should not be used for long text. Note that IBM Plex Sans Bold, ExtraLight, Text, and Thin are not for digital experiences.

Oops!

Bold, Medium, Text, Extralight and Thin should not be used in websites.

Italic

Each weight has an Italic style, which should only be used when you need to emphasize certain words in a sentence (titles of works, technical terms, names of devices, captions).

Case

Write content in sentence case. Title case can be used in proper titles, product names, and service names. but avoid all uppercase or “all caps.” Using sentence case is more comfortable for reading than uppercase because there is more distinction between the lowercase letters.

Oops!

Uppercase is not to be used for writing. It yells too much and is too formal.

Line length

Line lengths are built into our type code. Their max widths are based on type scale and character limit. The automatic line lengths are ~75 characters for body copy and ~50 characters for headlines.

Alignments

Text alignments aid in readability by organizing content for the eye to follow. We use this standard for Western Latin languages  to create clean and consistent layouts.

Oops!

Centered type does not fit the IBM typography approach, which creates relationships based on strong alignments and grid column systems.

Oops!

Flush right is reserved for languages that read right to left but is hard to read in Western Latin languages.

Oops!

Justified copy is difficult  to do well and can create uncomfortable spacing called a “river” that distracts the eye.

Fluid Type

Use fluid type to create a unified experience with the grid. Fluid type creates a cleaner type rag when you scale up the screen and provides a better line-length for reading. The fixed type scales are fluid between breakpoints to maintain the quality of typography.

Type Color

Careful consideration of color maintains quality and recognition for users. Legibility and accessibility are two keys. Keep colored type neutral when in paragraphs. Never use pure black (Hex value: #000000) for text, but instead, a range of grays. Use primary blue for primary actions.

Interactive text

Primary colors are used for text links or main actions.

Color type for specific usage

Example use cases for colored type include code snippet, warning, alert, etc.
